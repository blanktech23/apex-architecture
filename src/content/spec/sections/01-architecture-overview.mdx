│ ├── Agent Executor (7-9 agents per tenant │

│ │
│ ┌────────────┐ ┌────────────┐ │
│ │ NATS │ │ Valkey │ │
│ │ JetStream │ │ (BullMQ + │ │
│ │ (event bus)│ │ cache) │ │
│ └────────────┘ └────────────┘ │
│ NATS primary runs on VPS-1 only; VPS-2 connects │
│ as a leaf node. Known limitation: NATS JetStream │
│ runs standalone on the primary node (2 nodes │
│ cannot form Raft quorum). Valkey is a single │
│ managed instance. │
│ Both are acknowledged SPOFs — see Section 12 │
│ Dual-VPS Architecture for failure modes and │
│ mitigations. │
│ │
│ NATS Leaf Node Warnings: │
│ (a) Leaf nodes may accept publishes during hub │
│ unavailability without persisting to │
│ JetStream — messages silently lost. │
│ (b) JetStream acks do not guarantee durability │
│ (Jepsen 2024: writes flushed every 2 min). │
│ MITIGATION: Set sync_interval=always in │
│ JetStream config (forces fsync per write). │
│ Throughput drops to ~hundreds msg/s — │
│ acceptable for agent event bus volume. │
│ (c) All critical events use outbox pattern to │
│ guarantee delivery. │
