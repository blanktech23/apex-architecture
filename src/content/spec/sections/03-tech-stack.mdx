**Next.js App Router Conventions (KB-Enforced):**


- **Server/Client Component boundary:** Default to Server Components for all pages and data-fetching components. Only add `'use client'` for: form components with `useState`/`useFormState`, Socket.io real-time consumers (`useSocket` hook), React Flow interactive canvas, client-side search/filter controls, toast/modal providers. Explicit client component boundaries: `components/providers/*.tsx` (auth, socket, theme), `components/forms/*.tsx`, `components/realtime/*.tsx`, `components/charts/*.tsx` (Recharts).

> **Spec Amendment (CRITICAL — Attack Audit 2026-03-22):** NEVER add `'use client'` to any `layout.tsx` file. This cascades to ALL child pages, converting entire subtree to Client Components. Bundle size equals pre-RSC. Providers needing client state MUST be extracted to separate `'use client'` component imported into Server Component layout. Enforce via ESLint rule banning `'use client'` in `**/layout.tsx`.

> **Spec Amendment (Attack Audit 2026-03-22):** When Client Component wraps Server Component content, pass as `children` prop — never import Server Component directly inside `'use client'` file. Direct imports cascade the client directive to all imports.

- **Route protection:** ALL route protection MUST use Next.js `middleware.ts` at the edge --- NOT client-side route guards. Client-side guards flash protected content before redirect. Middleware intercepts at the edge before any HTML is sent. The `middleware.ts` performs ONLY lightweight, synchronous checks: cookie presence, JWT decode (without database calls), and redirect logic. It checks the better-auth session cookie, redirects unauthenticated users to `/login`, and handles tenant-scoped routing. **PPR preservation rule:** Do NOT perform database lookups in `middleware.ts` (e.g., checking `tenant_members.onboarding_completed`, role verification, feature flags). Database calls in middleware add ~150ms blocking latency before any HTML is sent, killing Partial Prerendering (PPR). All authorization checks that require database access (role verification, onboarding status, feature flags) MUST happen in Server Components behind granular `` boundaries, preserving PPR's ability to send the static shell immediately.

- **Suspense strategy:** Do NOT wrap the root layout in a single `` boundary (this blocks the entire page). Use granular `` boundaries around individual async Server Components (e.g., dashboard stats card, agent list, execution history table). Each boundary gets its own skeleton loader matching the component shape.

- **`loading.tsx` files:** Required for every route segment that fetches data: `/dashboard/loading.tsx`, `/escalations/loading.tsx`, `/settings/team/loading.tsx`, `/settings/billing/loading.tsx`, etc. Next.js uses these for instant loading states during navigation.

- **`redirect()` safety:** `redirect()` from `next/navigation` throws a `NEXT_REDIRECT` error internally. NEVER call `redirect()` inside a `try/catch` block --- it will be caught and swallowed. Either call `redirect()` outside `try/catch`, or use the `isRedirectError()` guard to re-throw redirect errors in catch blocks.

- **Data fetching cache control:** Server Components that call Supabase MUST specify explicit cache control. Use `export const revalidate = 0` (no cache) for tenant-scoped dynamic data. Use `export const revalidate = 3600` for static reference data. Never rely on Next.js default caching for authenticated, tenant-scoped queries.

- **`generateMetadata`:** Required on all public-facing pages (`/login`, `/forgot-password`, `/invite/*`, `/signup/*`, marketing pages) for SEO. Authenticated pages use a shared default metadata export.

- **`useSearchParams()`:** Must be wrapped in its own `` boundary in client components. Without this, it opts the entire page out of static generation. Wrap the component using `useSearchParams()` in a ``.

> **Spec Amendment (CRITICAL — Attack Audit 2026-03-22):** Next.js 15+/16 async request APIs: `params`, `searchParams`, `cookies()`, `headers()` are ALL async (return Promise). Always `await`: `const { id } = await params;`. Synchronous access returns Promise object, not values. Enforce via TypeScript — these are typed as `Promise<T>`.

> **Spec Amendment (CRITICAL — Attack Audit 2026-03-22):** Server Actions compile to public POST endpoints callable by anyone. Every Server Action MUST: (1) validate all inputs with Zod `safeParse`, (2) verify auth session inside the action (never assume middleware checked), (3) scope all DB operations to authenticated `tenant_id`. Return `{ error: FieldErrors | null }` pattern.

> **Spec Amendment (Attack Audit 2026-03-22):** `error.tsx` files MUST include `'use client'` directive (Next.js requirement). Never wrap root layout `{children}` in `<Suspense>` — catches `notFound()` and `redirect()`, returning HTTP 200 instead of 404/3xx.

> **Spec Amendment (Attack Audit 2026-03-22):** Server-to-Client props serialized via RSC protocol. `Date` → ISO string, `Map`/`Set` → `{}`, functions → error. Convert all TIMESTAMPTZ columns to ISO strings before passing to Client Components. Define `Serializable*` type layer using `string` for dates.

> **Spec Amendment (Attack Audit 2026-03-22):** Next.js has 4 caches (Request Memoization, Data Cache, Full Route Cache, Router Cache). Supabase JS client queries bypass Data Cache (not `fetch()` calls) — wrap in `unstable_cache()` with tags. On mutations, call BOTH `revalidatePath()` AND `revalidateTag()` (tag-only unreliable, #55960). `loading.tsx` cached 5 min by default — add `export const revalidate = 0` to dynamic segments.
