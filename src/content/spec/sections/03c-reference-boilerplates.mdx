### Recommended Build Path


**Step 1:** Fork **thorchh/agent-starter** -> instant chat UI + AI SDK streaming + tool calling ( human review -> client-facing document). The Bookkeeping Agent and Sales Agent should be designed with change order extensibility in mind --- specifically, the Sales Agent's pricing lookup tools and the Bookkeeping Agent's budget tracking should accept change-order-scoped queries.


**Lightweight-first principle:** thorchh/agent-starter gives 100% of the Vercel AI SDK tool-calling surface with zero framework overhead. Mastra templates are reference material for selective adoption --- not the base.
