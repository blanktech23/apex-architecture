Complete `CREATE TABLE` definitions for all core platform tables referenced in Section 5. Every tenant-scoped table includes RLS policies using the `(SELECT current_setting('app.tenant_id', true))::uuid` pattern with `FORCE ROW LEVEL SECURITY`. Platform-scoped tables are explicitly annotated with `NO RLS`.


**Convention alignment:** These DDL statements follow the same patterns established in the Business OS schema (Section 15.5) and K&B schema (Section 16.8): UUID PKs with `gen_random_uuid()`, snake_case plural table names, `TIMESTAMPTZ DEFAULT now()`, CHECK constraints for enum-like TEXT columns, and composite indexes on `(tenant_id, )`.
